문의게시판인 CS, 공지사항인 Notice를 추가했습니다.
CS 기능
1. 로그인을 하여 분점, 본점 구분을 합니다.
2. 로그인하면 localStorage에 username에 내가 직접 입력한 ID 값이 저장됩니다. 로그인창에서 admin_branch를 입력하면 localStorage에 admin_branch가 저장 되는 것 입니다.
3. 문의게시판 등록을 합니다. 그럼 작성자 ID 테이블을 백엔드의 서비스가 조회하여 그 ID에 맞는 이름을 가져옵니다. 그 이름을 response로 보내고 그걸 또 프론트엔드로 보내어 결국 "admin_branch"가 "분점관리자"로 바뀝니다.
4. 본점 관리자로 다시 로그인을 하여 분점 관리자가 보낸 문의사항을 상세보기버튼을 통해 답변을 작성하고 답변등록을 누르면 답변내용과 답변시간이 상세보기칸에 들어갑니다.
5. 다시 분점 관리자로 로그인하여 확인해보면 답변대기였던 상태가 답변완료로 변경되어 더 이상 게시물 수정을 하지 못합니다.

Notice 기능
1. 본점 관리자로 로그인 되어있을 때 등록을 할 수 있습니다.
2. 등록을 하고 상세보기를 누르면 수정 시간이 - 인데 이는 수정하고 상세보기를 누르면 수정시간이 표시되게 했습니다

CS, Notice 둘다 최근시간순으로 했고, 저번에 한 Promotion은 높은아이디순으로 게시글이 나오게 했습니다.


DB변경점
답변기능이 추가되었기 때문에 CS테이블에 답변과 답변시간이 추가되었고
Noitce테이블의 경우는 등록하면 수정시간도 같이 등록되기 때문에 
updateat not null을 삭제해서 등록해도 수정시간이 안들어가고 수정하고나서 
수정시간이 들어가게 변경했습니다
